<task type="auto">
  <name>Task 2: Move recipientKeys.find() inside try/catch in useSocket handler</name>
  <files>apps/client/src/hooks/useSocket.tsx</files>
  <action>
In `apps/client/src/hooks/useSocket.tsx`, the `onMessageCreated` handler at line 200 has `data.recipientKeys.find()` at line 205 OUTSIDE the try/catch block (which starts at line 208). If `data.recipientKeys` is undefined (e.g., from a malformed payload), the TypeError is unhandled and silently kills the handler.

Fix: Move the `recipientKeys.find()` call and the early-return guard inside the try/catch block.

Change from:
```typescript
const onMessageCreated = async (data: MessageEnvelope) => {
  if (data.senderId === user?.id) return;
  const myKey = data.recipientKeys.find((k) => k.recipientUserId === user?.id);
  if (!myKey) return;
  try {
    // ... decrypt and cache update
  } catch {
    // Decryption failed
  }
};
```

Change to:
```typescript
const onMessageCreated = async (data: MessageEnvelope) => {
  if (data.senderId === user?.id) return;
  try {
    const myKey = data.recipientKeys.find((k) => k.recipientUserId === user?.id);
    if (!myKey) return;
    // ... rest of decrypt and cache update stays the same
  } catch {
    // Decryption failed or malformed payload — skip; message will appear on next query refetch
  }
};
```

This is a minimal change — only the `const myKey` line and the `if (!myKey) return;` line move inside the existing try block. Everything else stays exactly where it is.
  </action>

  </verify>
  <done>
The `recipientKeys.find()` call is inside the try/catch block. A malformed or mismatched payload will be caught gracefully instead of crashing the handler with an unhandled TypeError.
