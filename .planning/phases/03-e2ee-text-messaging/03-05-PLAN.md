**1. Create `MessageItem.tsx`** — Single message row (per CONTEXT.md locked decisions):

- Discord-style flat list layout: left-aligned with avatar, username, timestamp
- Props: `message: DecryptedMessage`, `isGrouped: boolean` (whether to show compact/collapsed version), `isOwnMessage: boolean`, `onDelete: (id: string) => void`
- **Normal mode** (isGrouped = false): Shows avatar circle (colored with initials, like ServerIcon), display name, timestamp, message content, lock icon
- **Grouped mode** (isGrouped = true): Only shows message content with small timestamp on hover (no avatar/name) — same author within 5-minute window
- **Per-message lock icon**: Small lock SVG (🔒 without emoji — use an SVG path) next to timestamp, tooltip "End-to-end encrypted"
- **Hover toolbar**: Appears on message hover (absolute positioned, top-right). For own messages: "Delete" button (trash icon). For all: "Copy" button. Use `radix-ui` Tooltip for button labels.
- **Right-click context menu**: Use `radix-ui` ContextMenu wrapping the message row. Items: "Copy Text", "Delete Message" (only for own messages). Delete triggers confirmation.
- **Delete confirmation**: Use `radix-ui` AlertDialog. "Delete this message?" with Cancel and Delete buttons.
- **Message status indicators**: Show a clock icon for `status: 'pending'`, check icon for `status: 'sent'`, red retry icon for `status: 'failed'`
- **Decryption failure**: Show `[Decryption failed]` in red/muted text if `decryptionFailed` is true
- **Timestamp format**: Relative time for recent (e.g., "2 min ago"), full date/time on hover tooltip

Styling: dark theme consistent with existing app (zinc-900 bg, zinc-100 text, zinc-400 secondary text). Message hover bg: zinc-800/50.

**2. Create `MessageList.tsx`** — Scrollable container:

- Props: `channelId: string`, `channelName: string`
- Uses `useMessages(channelId)` for data
- Renders messages oldest-first (bottom = newest)
- **Time-window grouping**: Consecutive messages from same senderId within 5 minutes of each other → grouped (isGrouped = true for subsequent messages)
- **Infinite scroll up**: Sentinel `<div>` at the top of the scroll container observed by IntersectionObserver. When visible and `hasNextPage`, call `fetchNextPage()`. Show spinner while `isFetchingNextPage`.
- **Auto-scroll to bottom**: On initial load and when new messages arrive AND the user is already at the bottom. Use a ref to track scroll position.
- **Stay in place when scrolled up**: If user is not at bottom and new messages arrive, do NOT auto-scroll. Instead, increment a "new messages" counter.
- **Empty state**: "This is the beginning of #channel-name" with creation date, centered in the container
- **Loading state**: Show loading skeleton (3-5 placeholder message shapes with pulse animation)

**3. Create `NewMessagesButton.tsx`**:
- Floating button at the bottom of the message list when user is scrolled up and new messages have arrived
- Shows "X new messages" text
- Clicking scrolls to the bottom of the message list and resets counter
- Styled: semi-transparent dark background, fixed at bottom of scroll container, rounded pill shape

**4. Create `CryptoUnlockPrompt.tsx`**:
- Shows when crypto keys are not cached in the worker (after page reload)
- Full-area overlay within the channel view (not a modal over the whole app)
- Lock icon, text: "Enter your password to unlock encrypted messages"
- Password input field + "Unlock" button
- On submit: calls `loginDecrypt` from crypto.ts with the user's email + entered password + the stored encrypted key blob (fetched from /api/auth/me)
- On success: dismiss the prompt, messages decrypt normally
- On error: show "Incorrect password" inline error
- Uses existing `loginDecrypt` from crypto.ts which calls `LOGIN_DECRYPT` worker message

**1. Create `MessageInput.tsx`** — Auto-expanding textarea (per CONTEXT.md locked decisions):

- Props: `onSend: (plaintext: string) => void`, `disabled?: boolean`
- **Enter to send, Shift+Enter for new line**: `onKeyDown` handler checks `e.key === 'Enter' && !e.shiftKey` → prevent default, call onSend with trimmed text, clear textarea
- **Auto-expanding textarea**: Single-line height by default. On input, set `style.height = 'auto'` then `style.height = Math.min(scrollHeight, MAX_HEIGHT)px`. MAX_HEIGHT ≈ 5 lines (~120px). After that, textarea scrolls internally.
- **Reset height** after sending (clear content and reset to single line)
- **Disabled state**: When `disabled` is true (e.g., crypto not unlocked), show placeholder "Unlock encryption to send messages" and disable input
- **Styling**: Dark input bar at the bottom of the channel view. zinc-800 bg, rounded-lg, padding, placeholder text "Message #channel-name"
- **Empty message prevention**: Don't send if text is empty/whitespace-only after trim

**2. Create `ChannelView.tsx`** — Route component for `/servers/:serverId/channels/:channelId`:

- Gets `channelId` and `serverId` from `useParams()`
- Gets channel info from `useChannels(serverId)` — find the channel by ID for name/topic
- Checks if crypto is unlocked (add a check — e.g., try a simple worker ping or use a state flag). If not unlocked, show `CryptoUnlockPrompt` instead of messages.
- **Layout**: Flex column, full height:
  - **Header bar** (h-12, border-b): Hash icon + channel name + topic (if any). Consistent with existing content header pattern from Phase 2.
  - **MessageList** (flex-1, overflow-y-auto): The scrollable message area
  - **MessageInput** (bottom, border-t): The input bar
- **Send handler**: When MessageInput fires onSend:
  - Get current server members from useServerMembers(serverId) — extract `{ userId, x25519PublicKey }` for each member (including self)
  - Call `useSendMessage(channelId).mutate({ plaintext, recipients })`
- Subscribes to the channel room on mount: `socket.emit('channel:subscribe', { channelId })` (ensures real-time messages are received even if socket connected before this channel was navigated to)
