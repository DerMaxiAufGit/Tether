**1. Update Socket.IO message:created handler** in `useSocket.tsx`:

When a `message:created` event arrives for a DM channel, also invalidate the `['dms']` query so the DM list re-sorts by most recent activity:

```typescript
// Inside the existing onMessageCreated handler:
// After appending the message to the channel's query cache...
// Also update DM list sort order
void queryClient.invalidateQueries({ queryKey: ['dms'] });
```

This ensures that when a new DM message arrives, the DM conversation list moves that conversation to the top.

**2. Handle socket reconnection** — After reconnect, invalidate message queries for the currently active channel to fetch any messages missed during disconnection:

In the `useSocket.tsx` provider, add a `reconnect` event listener:
```typescript
const onReconnect = () => {
  // Invalidate all message queries so the currently viewed channel refetches
  void queryClient.invalidateQueries({ queryKey: ['messages'] });
  // Also refresh DM list
  void queryClient.invalidateQueries({ queryKey: ['dms'] });
};
socket.io.on("reconnect", onReconnect);
// Clean up: socket.io.off("reconnect", onReconnect);
```

This ensures messages sent while the user was offline appear after reconnection.

**3. Ensure channel:subscribe re-emitted on reconnect** — The socket loses room memberships on disconnect. After reconnect, the server-side `registerConnectionHandlers` re-runs and re-joins all rooms. Verify this is the case (it should be, since `registerConnectionHandlers` runs on every connection including reconnections). If not, add a `connect` event handler on the client that emits `channel:subscribe` for the currently viewed channel.
