    3. Create `apps/client/src/hooks/useSocket.ts`:
       - Create a React context + provider (`SocketProvider`) that manages a singleton `socket.io-client` instance
       - The socket connects when the user is authenticated (read access token from `getAccessToken()` in `api.ts`)
       - The socket disconnects on logout (expose a `disconnect` method or react to auth state)
       - Socket.IO client config: `{ auth: { token }, transports: ["websocket"], autoConnect: false }`
       - Connect manually in a useEffect when auth state changes
       - On connect, register Socket.IO event listeners that invalidate TanStack Query cache:
         - `server:created` -> `invalidateQueries({ queryKey: ["servers"] })`
         - `server:deleted` -> `invalidateQueries({ queryKey: ["servers"] })`
         - `server:updated` -> `invalidateQueries({ queryKey: ["servers"] })`
         - `member:joined` -> `invalidateQueries({ queryKey: ["servers", data.serverId, "members"] })`
         - `member:left` -> `invalidateQueries({ queryKey: ["servers", data.serverId, "members"] })` AND `invalidateQueries({ queryKey: ["servers"] })` (user might have been kicked)
         - `channel:created` / `channel:updated` / `channel:deleted` -> `invalidateQueries({ queryKey: ["servers", data.serverId, "channels"] })`
       - Clean up all listeners on unmount with exact function references for `socket.off()`
       - Export `useSocket()` hook that returns the socket instance
       - Import `io` from `socket.io-client` (already installed as a dependency of the server — verify it's in client package.json, if not, install `socket.io-client`)

       IMPORTANT: The Socket.IO client library is `socket.io-client`, not `socket.io`. Check if it's already installed in the client. If not, add it:
       ```bash
       pnpm --filter @tether/client add socket.io-client
       ```

       IMPORTANT: Register event listeners ONCE at the SocketProvider level, not in individual components. Use stable function references in the useEffect to avoid duplicate listeners (React StrictMode pitfall from RESEARCH.md).

    4. Create `apps/client/src/hooks/useServers.ts`:
       ```typescript
       import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
       import { api } from "@/lib/api";
       import type { ServerResponse, CreateServerRequest } from "@tether/shared";

       export function useServers() {
         return useQuery({
           queryKey: ["servers"],
           queryFn: () => api.get<ServerResponse[]>("/api/servers"),
         });
       }

       export function useCreateServer() {
         const queryClient = useQueryClient();
         return useMutation({
           mutationFn: (data: CreateServerRequest) =>
             api.post<ServerResponse>("/api/servers", data),
           onSuccess: () => {
             queryClient.invalidateQueries({ queryKey: ["servers"] });
           },
         });
       }
       ```
