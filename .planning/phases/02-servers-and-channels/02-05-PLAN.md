       - Props: `channel: ChannelResponse`, `isSelected: boolean`
       - Uses `useSortable({ id: channel.id })` from `@dnd-kit/sortable`
       - Renders: icon (# for text, speaker/volume icon for voice) + channel name
       - Style: `transform: CSS.Transform.toString(transform)`, `transition`, `opacity: isDragging ? 0.5 : 1`
       - Selected state: lighter background (bg-zinc-700/50), text highlight
       - Hover state: subtle background change
       - Wrapped in `<Link>` to `/servers/${channel.serverId}/channels/${channel.id}` (for text channels)
       - Drag handle: the entire item is the drag handle (attributes + listeners spread on the outer div)

    2. Create `apps/client/src/components/server/ChannelList.tsx`:
       - Props: `serverId: string`
       - Uses `useChannels(serverId)` to get channel data
       - Uses `useReorderChannels(serverId)` for the reorder mutation
       - Groups channels by type: "Text Channels" group and "Voice Channels" group
       - Each group has a collapsible header (click to toggle visibility). Use local state `useState` for collapsed state per group.
       - Group header: small uppercase text (text-xs, text-zinc-400, font-semibold), with a chevron icon that rotates when collapsed
       - Inside each group: `DndContext` + `SortableContext` wrapping `ChannelItem` components
         - `sensors`: PointerSensor + KeyboardSensor (with sortableKeyboardCoordinates)
         - `collisionDetection`: closestCenter
         - `onDragEnd`: calculate new positions using `arrayMove`, optimistically update local state, then call reorder mutation
       - Local state for optimistic reorder: `const [localChannels, setLocalChannels] = useState(channels)` — sync with server data via useEffect when `channels` changes
       - Server name header at the top of the channel panel: clickable, shows server name in bold, with a dropdown chevron (dropdown menu for future settings — for now just show the header without dropdown functionality, or add a minimal dropdown with "Invite People" and "Server Settings" options)
       - User info bar at the very bottom: display current user's avatar (colored initial circle), display name, and a settings gear icon. Get user data from `useAuth()`. Settings gear navigates to `/change-password` for now.

       IMPORTANT: Reorder is per-type group. When the user drags a text channel, only text channels are reordered (voice channels keep their positions). Send the full position mapping for ALL channels (both types) to the server. The simplest approach: after drag end in one group, recalculate all positions — text channels get positions 0..N-1, voice channels get positions N..N+M-1. This keeps everything consistent.

       IMPORTANT: Follow RESEARCH.md Pitfall 2 — always send the full ordered list to the reorder endpoint. Don't try to patch individual positions.
