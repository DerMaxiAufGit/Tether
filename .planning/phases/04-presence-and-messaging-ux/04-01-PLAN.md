**1. Create `apps/server/src/socket/handlers/presence.ts`:**

This module handles:
- `onConnect`: Increment Redis `presence:count:{userId}`, broadcast `presence:update` if count becomes 1 (first socket), send `presence:snapshot` of all server members' statuses to the connecting socket
- `onDisconnect`: After 30-second grace timeout, decrement Redis `presence:count:{userId}`, broadcast `presence:update` if count reaches 0 (last socket)
- `presence:idle` event: Set `presence:idle:{userId}` in Redis (with 15-minute TTL), broadcast `presence:update` with status `idle` to server rooms
- `presence:active` event: Delete `presence:idle:{userId}`, broadcast `presence:update` with status resolved (check DND first, then online)
- `presence:dnd` event: Toggle `presence:dnd:{userId}` in Redis (SET or DEL), broadcast `presence:update`

Status resolution logic (pure function, exported for testing):
```typescript
export function resolveStatus(count: number, hasDnd: boolean, hasIdle: boolean): PresenceStatus {
  if (count <= 0) return "offline";
  if (hasDnd) return "dnd";
  if (hasIdle) return "idle";
  return "online";
}
```

For presence:snapshot on connect, the handler needs the list of `memberships` (server IDs the user belongs to). It should:
1. Get all unique user IDs across all server rooms the connecting user is a member of (query `serverMembers` table grouped by serverId)
2. For each unique userId, read Redis keys: `presence:count:{userId}`, `presence:dnd:{userId}`, `presence:idle:{userId}`
3. Use pipeline for efficiency (redis.multi() or individual gets â€” pipeline is better)
4. Build a `Record<string, PresenceStatus>` map and emit `presence:snapshot` to the connecting socket only (not broadcast)

Import the Redis client from `../../db/redis.js`. Import presence types from `@tether/shared`.

The function signature should be:
```typescript
export async function registerPresenceHandlers(
  socket: Socket,
  io: SocketIOServer,
  logger: FastifyBaseLogger,
  memberships: Array<{ serverId: string }>,
): Promise<void>
```

The `io` parameter is needed to broadcast to rooms (socket.to only sends to others in the room, which is correct for broadcasts; use io.to for including the sender when needed for snapshot).

For the grace period, use `setTimeout(async () => { ... }, 30_000)`. Inside the timeout:
- Decrement `presence:count:{userId}`
- If result <= 0: delete the key, delete idle key, broadcast offline to all server rooms
- To broadcast to all server rooms: query memberships again inside the timeout (user could have joined/left servers during the 30s)
