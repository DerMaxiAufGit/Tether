<objective>
Build the complete auth REST API: register, login, logout, refresh, and change-password endpoints. Registration stores Argon2id-hashed auth key and encrypted key blobs. Login verifies and returns a JWT access token + refresh cookie + key bundle. Refresh rotates tokens atomically. Change-password re-encrypts key blobs in a DB transaction.

Purpose: This is the server-side auth backbone. Every API call in every future phase is authenticated via the JWT middleware built here. The zero-knowledge guarantee depends on this layer only storing hashed auth keys and encrypted blobs.
Output: Five auth endpoints, JWT utility, auth middleware plugin, and shared auth types.
</objective>

  <action>
    All routes are Fastify route plugins. Each file exports a default async function that receives the Fastify instance and registers routes.

    **POST /api/auth/register** (register.ts):
    1. Parse body as RegisterRequest (validate: email format, all required fields present, displayName 1-50 chars)
    2. Check email uniqueness — if exists, return 409 { error: "Email already registered" }
    3. Decode authKey from base64 to Buffer
    4. Hash authKey with Argon2id: `argon2.hash(authKeyBuffer, { type: argon2.argon2id, memoryCost: 65536, timeCost: 3, parallelism: 4 })`
    5. Decode all base64 fields (salt, public keys, encrypted private keys, IVs) to Buffers
    6. Insert into users table with all fields
    7. Generate refresh token jti (crypto.randomUUID()), insert into refresh_tokens with 7-day expiry
    8. Sign access token + refresh token
    9. Set refresh token as httpOnly, SameSite=Lax, Secure (in production), Path=/api/auth/refresh cookie
    10. Return 201 { accessToken, user: { id, email, displayName } }

    **POST /api/auth/login** (login.ts):
    1. Parse body as LoginRequest (validate: email, authKey present)
    2. Find user by email — if not found, return 401 { error: "Invalid credentials" } (do NOT reveal whether email exists)
    3. Decode authKey from base64 to Buffer
    4. Verify with argon2.verify(user.authKeyHash, authKeyBuffer) — if fails, return 401 { error: "Invalid credentials" }
    5. Generate refresh token jti, insert into refresh_tokens
    6. Sign access + refresh tokens
    7. Set refresh cookie (same settings as register)
    8. Return 200 LoginResponse with accessToken, user info, and full keyBundle (salt, public keys, encrypted private keys, IVs — all as base64)
    9. Convert bytea Buffer values from DB to base64 for the response

    **POST /api/auth/logout** (logout.ts):
    1. Read refresh token from cookie
    2. If present, verify it, extract jti, delete from refresh_tokens table
    3. Clear the refresh cookie
    4. Return 200 { success: true }
    5. Do NOT require access token — user should be able to log out even with expired access token

    **POST /api/auth/refresh** (refresh.ts):
    1. Read refresh token from cookie — if missing, return 401
    2. Verify refresh token — if expired/invalid, clear cookie, return 401
    3. Extract jti from token payload
    4. In a DB transaction with SELECT ... FOR UPDATE:
       a. Find refresh_tokens row by jti
       b. If not found (already used/rotated), this is a replay attack: delete ALL refresh tokens for this user, clear cookie, return 401 { error: "Token reuse detected" }
       c. Delete the current refresh token row
       d. Insert new refresh token with new jti and 7-day expiry
    5. Sign new access token + new refresh token
    6. Set new refresh cookie
    7. Return 200 { accessToken }

    **POST /api/auth/change-password** (change-password.ts):
    1. Requires authentication (use auth preHandler)
    2. Parse body as ChangePasswordRequest
    3. In a DB transaction with SELECT ... FOR UPDATE on user row:
       a. Verify oldAuthKey against current authKeyHash with argon2.verify — if fails, return 401
       b. Hash newAuthKey with Argon2id
       c. Decode all new base64 fields to Buffers
       d. Update user: authKeyHash, kdfSalt (newSalt), all encrypted private key fields, all IV fields, updatedAt
       e. Delete ALL refresh_tokens for this user (force re-login on all devices)
    4. Return 200 { success: true, message: "Password changed. Please log in again." }

    IMPORTANT:
    - Use `Buffer.from(base64String, "base64")` for base64 -> Buffer conversion
    - Use `buffer.toString("base64")` for Buffer -> base64 conversion
    - All error responses use consistent format: { error: string }
    - All routes should have Fastify schema validation for request body where appropriate
