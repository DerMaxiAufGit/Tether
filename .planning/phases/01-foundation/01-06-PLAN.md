  <action>
    1. Create docker-compose.yml with two networks and six services:

    **Networks:**
    - `internal`: bridge — used by app, postgres, redis, minio
    - `coturn_external`: bridge — used ONLY by coturn (internet-facing, cannot reach internal)

    **Services:**

    a) **postgres** (PostgreSQL 17):
       - image: postgres:17-alpine
       - networks: [internal]
       - environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB (from .env)
       - volumes: postgres_data:/var/lib/postgresql/data
       - healthcheck: pg_isready -U $${POSTGRES_USER} -d $${POSTGRES_DB}, interval 10s, retries 5, start_period 30s, timeout 10s
       - ports: "5432:5432" (for local dev access; remove in production)

    b) **redis** (Redis 7 Alpine):
       - image: redis:7-alpine
       - networks: [internal]
       - command: redis-server --requirepass ${REDIS_PASSWORD}
       - healthcheck: redis-cli -a ${REDIS_PASSWORD} ping, interval 10s, retries 5
       - ports: "6379:6379" (for local dev access)

    c) **minio** (MinIO):
       - image: minio/minio
       - networks: [internal]
       - environment: MINIO_ROOT_USER, MINIO_ROOT_PASSWORD
       - command: server /data --console-address ":9001"
       - volumes: minio_data:/data
       - ports: "9000:9000" (API), "9001:9001" (console)
       - healthcheck: curl -f http://localhost:9000/minio/health/live, interval 10s, retries 5

    d) **coturn** (Coturn):
       - image: coturn/coturn:latest
       - networks: [coturn_external] (NOT on internal)
       - ports: "3478:3478" (STUN), "3478:3478/udp", "5349:5349" (TURNS), "5349:5349/udp", "49152-49200:49152-49200/udp" (TURN relay range — intentionally small for dev; production uses larger range or host networking)
       - volumes: ./coturn/turnserver.conf:/etc/coturn/turnserver.conf:ro
       - restart: unless-stopped
       - Comment in the file: "For production, consider network_mode: host instead of port ranges. On Mac/Windows Docker Desktop, host networking doesn't work — use port ranges."

    e) **app** (Tether server):
       - build: . (uses Dockerfile)
       - networks: [internal]
       - ports: "3001:3001"
       - depends_on: postgres (service_healthy), redis (service_healthy)
       - environment: DATABASE_URL, REDIS_URL, JWT_SECRET, JWT_REFRESH_SECRET, COOKIE_SECRET, CLIENT_URL, PORT=3001
       - env_file: .env
       - restart: unless-stopped

    **Volumes:**
    - postgres_data
    - minio_data

       ```
       # JWT (generate with: openssl rand -hex 32)
       JWT_SECRET=changeme
       JWT_REFRESH_SECRET=changeme

       # Cookie

       # Coturn
       COTURN_SECRET=changeme
       COTURN_REALM=tether.local

       # MinIO

       MINIO_ROOT_PASSWORD=changeme

       # App
       CLIENT_URL=http://localhost:5173
       PORT=3001
       ```

    3. Create coturn/turnserver.conf:
       ```
       # Tether Coturn Configuration
       # SECURITY: Block all private/reserved IP ranges to prevent TURN relay abuse
       denied-peer-ip=10.0.0.0-10.255.255.255
       denied-peer-ip=172.16.0.0-172.31.255.255
       denied-peer-ip=192.168.0.0-192.168.255.255
       denied-peer-ip=127.0.0.0-127.255.255.255
       denied-peer-ip=169.254.0.0-169.254.255.255
       denied-peer-ip=::1

       # Authentication via shared secret (ephemeral HMAC credentials)
       use-auth-secret
       static-auth-secret=${COTURN_SECRET}
       realm=${COTURN_REALM}

       # Security hardening
       fingerprint
       no-multicast-peers
       no-cli

       # Relay port range (small for dev, expand for production)
       min-port=49152
       max-port=49200

       # Logging
       log-file=stdout
       verbose
       ```

  <files>

  <action>
    1. Create Dockerfile (multi-stage build for the app server):
       ```dockerfile
       # Stage 1: Install dependencies
       FROM node:20-alpine AS deps
       RUN corepack enable && corepack prepare pnpm@9 --activate
       WORKDIR /app
       COPY pnpm-lock.yaml pnpm-workspace.yaml package.json ./
       COPY apps/server/package.json apps/server/
       COPY packages/shared/package.json packages/shared/
       RUN pnpm install --frozen-lockfile --filter @tether/server --filter @tether/shared

       # Stage 2: Build
       FROM node:20-alpine AS builder
       RUN corepack enable && corepack prepare pnpm@9 --activate
       WORKDIR /app
       COPY --from=deps /app/node_modules ./node_modules
       COPY --from=deps /app/apps/server/node_modules ./apps/server/node_modules
       COPY --from=deps /app/packages/shared/node_modules ./packages/shared/node_modules 2>/dev/null || true
       COPY . .
       RUN pnpm --filter @tether/shared build 2>/dev/null || true
       RUN pnpm --filter @tether/server build

       # Stage 3: Production
       FROM node:20-alpine AS runner
       RUN corepack enable && corepack prepare pnpm@9 --activate
       WORKDIR /app
       ENV NODE_ENV=production
       COPY --from=builder /app/apps/server/dist ./apps/server/dist
       COPY --from=builder /app/apps/server/package.json ./apps/server/
       COPY --from=builder /app/packages/shared ./packages/shared
       COPY --from=builder /app/node_modules ./node_modules
       COPY --from=builder /app/pnpm-workspace.yaml ./
       COPY --from=builder /app/package.json ./
       EXPOSE 3001
       CMD ["node", "apps/server/dist/index.js"]
       ```
       NOTE: This Dockerfile may need adjustments once the build pipeline is fully working. The key structure is correct: deps -> build -> minimal runtime image. Adjust COPY paths based on actual build outputs.

    2. Create .dockerignore:
       ```
       node_modules
       .git
       .planning
       .turbo
       dist
       *.md
       .env
       .env.*
       !.env.example
       ```

    3. Create scripts/generate-secrets.sh:
       ```bash
       #!/usr/bin/env bash
       set -euo pipefail

       ENV_FILE=".env"

       if [ -f "$ENV_FILE" ]; then
         echo ".env already exists. Remove it first to regenerate."
         exit 1
       fi

       echo "Generating secrets..."

       POSTGRES_PASSWORD=$(openssl rand -hex 16)
       REDIS_PASSWORD=$(openssl rand -hex 16)
       JWT_SECRET=$(openssl rand -hex 32)
       JWT_REFRESH_SECRET=$(openssl rand -hex 32)
       COOKIE_SECRET=$(openssl rand -hex 32)
       COTURN_SECRET=$(openssl rand -hex 32)
       MINIO_ROOT_PASSWORD=$(openssl rand -hex 16)

       cat > "$ENV_FILE" <<EOF
       # Generated by scripts/generate-secrets.sh on $(date -u +"%Y-%m-%dT%H:%M:%SZ")
       # DO NOT COMMIT THIS FILE

       # PostgreSQL
       POSTGRES_USER=tether
       POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
       POSTGRES_DB=tether
       DATABASE_URL=postgresql://tether:${POSTGRES_PASSWORD}@postgres:5432/tether

       # Redis
       REDIS_PASSWORD=${REDIS_PASSWORD}
       REDIS_URL=redis://:${REDIS_PASSWORD}@redis:6379

       # JWT
       JWT_SECRET=${JWT_SECRET}
       JWT_REFRESH_SECRET=${JWT_REFRESH_SECRET}

       # Cookie
       COOKIE_SECRET=${COOKIE_SECRET}

       # Coturn
       COTURN_SECRET=${COTURN_SECRET}
       COTURN_REALM=tether.local

       # MinIO
       MINIO_ROOT_USER=minioadmin
       MINIO_ROOT_PASSWORD=${MINIO_ROOT_PASSWORD}

       # App
       CLIENT_URL=http://localhost:5173
       PORT=3001
       EOF

       echo "Generated $ENV_FILE with random secrets."
       echo "Review and customize as needed."
       ```

    4. Make the script executable: `chmod +x scripts/generate-secrets.sh`
  </action>
  <verify>

  </verify>
  <done>Multi-stage Dockerfile for production server image. Secrets generation script creates .env with cryptographically random secrets via openssl. .dockerignore prevents bloated images.</done>
