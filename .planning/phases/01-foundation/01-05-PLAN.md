  <action>
    1. Install Socket.IO dependencies in apps/server:
       `pnpm add socket.io @socket.io/redis-streams-adapter redis`

    2. Create apps/server/src/socket/middleware/auth.ts:
       - Export a Socket.IO middleware function: `(socket, next) => { ... }`
       - Extract token from `socket.handshake.auth?.token`
       - If no token, call `next(new Error("Authentication required"))`
       - Verify token with `verifyAccessToken` from lib/jwt.ts
       - On success: set `socket.data.userId = payload.sub` and call `next()`
       - On failure: call `next(new Error("Invalid or expired token"))`
       - Augment Socket.IO types: extend `SocketData` interface with `{ userId: string }`

    3. Create apps/server/src/socket/handlers/connection.ts:
       - Export a function that registers event handlers on a connected socket
       - On connection: log `"User ${socket.data.userId} connected"` (use Fastify logger if available, otherwise console.log)
       - On disconnect: log `"User ${socket.data.userId} disconnected"`
       - Register a "ping" event that responds with "pong" — this is a basic health check for WebSocket connectivity
       - Future phases will add handlers here (message:send, typing:start, voice:join, etc.)

    4. Create apps/server/src/socket/index.ts:
       - Export a function `setupSocketIO(httpServer: HttpServer, logger: FastifyBaseLogger)` that:
         a. Creates a Redis client: `createClient({ url: process.env.REDIS_URL || "redis://localhost:6379" })`
         b. Connects the Redis client (await redisClient.connect())
         c. Creates Socket.IO server: `new Server(httpServer, { adapter: createAdapter(redisClient), cors: { origin: process.env.CLIENT_URL || "http://localhost:5173", credentials: true } })`
         d. Registers the auth middleware via `io.use(socketAuthMiddleware)`
         e. Registers connection handler: `io.on("connection", (socket) => registerConnectionHandlers(socket, logger))`
         f. Returns the io instance (for use in route handlers that need to emit)
       - Handle Redis connection errors gracefully (log warning, Socket.IO still works without adapter in dev)

    5. Update apps/server/src/index.ts:
       - After creating the Fastify instance and registering plugins, get the underlying HTTP server
       - Call `setupSocketIO(server.server, server.log)` after server.listen()
       - Store the io instance on the Fastify instance (decorate) for route handlers to access: `fastify.decorate("io", io)`
       - Augment Fastify types: declare module "fastify" { interface FastifyInstance { io: Server } }
       - On server close, disconnect Redis client and close Socket.IO

    IMPORTANT:
    - Use `socket.handshake.auth.token`, NOT `extraHeaders` — extraHeaders don't work with pure WebSocket transport
    - Use `@socket.io/redis-streams-adapter` (NOT `@socket.io/redis-adapter`) — Streams adapter handles Redis disconnection without packet loss
    - The Redis client for Socket.IO should be separate from any future Redis usage (e.g., caching) to avoid blocking
  </action>
