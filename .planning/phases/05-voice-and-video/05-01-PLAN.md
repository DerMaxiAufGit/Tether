Create `apps/server/src/routes/voice/index.ts` as a Fastify route plugin following the established pattern from auth/servers/channels routes:

1. **Credential generation function** using Node.js built-in `crypto`:
   ```typescript
   import crypto from "node:crypto";

   function generateTurnCredentials(userId: string) {
     const secret = process.env.COTURN_SECRET;
     if (!secret) throw new Error("COTURN_SECRET not configured");
     const ttl = 86400; // 24 hours
     const expiry = Math.floor(Date.now() / 1000) + ttl;
     const username = `${expiry}:${userId}`;
     const credential = crypto.createHmac("sha1", secret).update(username).digest("base64");
     return { username, credential, ttl };
   }
   ```

2. **GET /api/voice/turn-credentials** (auth-protected):
   - Uses `server.addHook("preHandler", server.auth)` for JWT auth (same pattern as servers/channels routes)
   - Generates ephemeral credentials for `request.user.id`
   - Returns `TurnCredentialsResponse`:
     ```json
     {
       "iceServers": [
         { "urls": "stun:localhost:3478" },
         { "urls": "turn:localhost:3478", "username": "...", "credential": "..." },
         { "urls": "turns:localhost:5349", "username": "...", "credential": "..." }
       ]
     }
     ```
   - The host should be read from `process.env.COTURN_HOST ?? "localhost"` so it's configurable for production.
   - Add `COTURN_HOST` to `.env.example` with comment.

3. **Register the route** in `apps/server/src/index.ts`:
   ```typescript
   await server.register(import("./routes/voice/index.js"), { prefix: "/api/voice" });
   ```
   Place the registration alongside the existing route registrations.
