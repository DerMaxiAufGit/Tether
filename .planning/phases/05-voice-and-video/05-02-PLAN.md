  <name>Task 1: Create voice signaling socket handlers with Redis participant tracking</name>
  <files>apps/server/src/socket/handlers/voice.ts</files>
  <action>
Create `apps/server/src/socket/handlers/voice.ts` following the established handler pattern from `typing.ts` and `presence.ts`:

**Redis data model:**
- `voice:participants:{channelId}` — Redis Set of userId strings (same pattern as typing)
- `voice:channel:{userId}` — Redis String storing the channelId the user is currently in (for single-channel enforcement and disconnect cleanup)
- No TTL (voice sessions can last hours; cleanup is event-driven)

**Handler: `registerVoiceHandlers(socket, io, logger)`**

1. **voice:join** `(VoiceJoinPayload)`:
   - Verify channel exists and is type "voice" (query DB: `channels` table where `id = channelId AND type = 'voice'`)
   - Verify user is a member of the channel's server (query DB: `serverMembers` where `serverId` and `userId`)
   - Check if user is already in a different voice channel (`GET voice:channel:{userId}`). If so, auto-leave the previous channel first (emit internal leave logic: SREM from old channel set, leave old voice room, broadcast `voice:participant_left` to old channel)
   - `SADD voice:participants:{channelId} userId`
   - `SET voice:channel:{userId} channelId`
   - `socket.join(voice:${channelId})`
   - Fetch current participant list from Redis Set (`SMEMBERS`), then for each userId query the `users` table to get `displayName` — build `VoiceParticipant[]` with all flags defaulting to `{ muted: false, deafened: false, cameraOn: false, speaking: false, screenShareCount: 0 }`
   - Emit `voice:joined` to the joining socket with the participant list
   - Broadcast `voice:participant_joined` to `voice:${channelId}` room (except sender) with the new participant object
   - Also broadcast to `server:${serverId}` room: `voice:channel_update` with `{ channelId, participantCount }` so the channel list sidebar can show who's in the voice channel

2. **voice:leave** `(VoiceJoinPayload)`:
   - `SREM voice:participants:{channelId} userId`
   - `DEL voice:channel:{userId}`
   - `socket.leave(voice:${channelId})`
   - Broadcast `voice:participant_left` to `voice:${channelId}` room with `{ channelId, userId }`
   - Broadcast `voice:channel_update` to `server:${serverId}` room with updated count

3. **voice:offer** `(VoiceSignalPayload)`:
   - Find target socket(s) in `user:${to}` room
   - Relay the full payload to target with `from: userId` added

4. **voice:answer** `(VoiceSignalPayload)`:
   - Same relay pattern as voice:offer

5. **voice:ice** `(VoiceIcePayload)`:
   - Same relay pattern as voice:offer/answer

6. **voice:mute** `(VoiceMutePayload)`:
   - Broadcast to `voice:${channelId}` room: `voice:mute` with `{ userId, muted }`

7. **voice:deafen** `(VoiceDeafenPayload)`:
   - Broadcast to `voice:${channelId}` room: `voice:deafen` with `{ userId, deafened }`

8. **voice:camera** `(VoiceCameraPayload)`:
   - Broadcast to `voice:${channelId}` room: `voice:camera` with `{ userId, cameraOn }`

9. **voice:screen_share** `(VoiceScreenSharePayload)`:
   - Broadcast to `voice:${channelId}` room: `voice:screen_share` with `{ userId, screenShareCount, streamId, action }`

10. **voice:speaking** `(VoiceSpeakingPayload)`:
    - Broadcast to `voice:${channelId}` room (except sender): `voice:speaking` with `{ userId, speaking }`

11. **disconnect** handler:
    - Read `voice:channel:{userId}` to find current voice channel
    - If in a channel: execute leave logic (SREM, DEL, broadcast `voice:participant_left` and `voice:channel_update`)
    - Pattern: same as typing.ts disconnect cleanup iterating socket.rooms

**Relay pattern for offer/answer/ice:**
```typescript
// Find all sockets for the target user in any room, emit to them
io.to(`user:${payload.to}`).emit(eventName, { ...payload, from: userId });
```

Error handling: wrap each handler in try/catch with logger.error (same pattern as typing.ts).
