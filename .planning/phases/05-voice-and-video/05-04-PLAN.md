**Parameters:**
```typescript
interface UseVoiceActivityOptions {
  stream: MediaStream | null;
  enabled: boolean; // false when muted — stop broadcasting "speaking" when mic is muted
  onSpeakingChange: (speaking: boolean) => void;
}
```

**Implementation:**
1. Create `AudioContext` and `AnalyserNode` in a useEffect when `stream` is provided and `enabled` is true
2. Call `audioCtx.resume()` inside the effect (user gesture already happened from the join click — resolves Pitfall 6 from research)
3. Connect `MediaStreamSource` from the stream to the AnalyserNode
4. Start a `requestAnimationFrame` loop:
   ```typescript
   const THRESHOLD = 0.015; // tunable; Claude's discretion per CONTEXT.md
   const HYSTERESIS_MS = 150; // debounce to prevent flicker
   let speaking = false;
   let lastChange = 0;

   function tick() {
     analyser.getByteFrequencyData(data);
     const rms = Math.sqrt(
       data.reduce((sum, v) => sum + (v / 255) ** 2, 0) / data.length
     );
     const now = Date.now();
     const newSpeaking = rms > THRESHOLD;
     if (newSpeaking !== speaking && now - lastChange > HYSTERESIS_MS) {
       speaking = newSpeaking;
       lastChange = now;
       onSpeakingChange(speaking);
     }
     rafId = requestAnimationFrame(tick);
   }
   ```
5. Cleanup: `cancelAnimationFrame(rafId)`, disconnect source, close AudioContext

**When `enabled` is false (user muted):**
- Stop the RAF loop
- If currently speaking, fire `onSpeakingChange(false)` to clear the indicator
- Do NOT close the AudioContext (recreating is expensive) — just stop the loop

**Return:** `{ isSpeaking: boolean }` (local state for self-view indicator)
