**ParticipantTile.tsx:**
Renders a single participant in the voice channel grid.

Props:
```typescript
interface ParticipantTileProps {
  participant: VoiceParticipant;
  stream: MediaStream | null; // remote audio/video stream (null = avatar only)
  isScreenShare?: boolean; // if true, renders as a screen share tile (larger)
  isSelf?: boolean; // subtle "You" badge
}
```

Layout (per locked decisions):
- Circular/rounded avatar tile with name underneath
- Avatar: colored circle with initial letter (same pattern as UserInfoBar avatar)
- When `participant.cameraOn && stream has video track`: seamlessly show `<video>` element with `autoPlay muted playsInline` (muted for self-view, srcObject = stream). The video fills the tile area with `object-fit: cover` in a rounded container.
- **Speaking indicator (locked decision):** Pulsing avatar animation COMBINED with green border glow:
  - Green `ring-2 ring-emerald-400` border when `participant.speaking`
  - CSS `animate-pulse-speak` animation on the avatar container (defined in index.css)
  - The animation should be a subtle scale pulse (1.0 → 1.05 → 1.0) at ~1.5s duration
- **Mute/deafen/camera-off badges (locked decision):** Small icon badges overlaid on tile corners (bottom-right corner stack):
  - Muted: red microphone-off icon
  - Deafened: red headphone-off icon
  - Camera off: (no badge — avatar is the default state)
- **Self indicator:** Small "You" text badge at top-left when `isSelf`
- **Screen share tile:** When `isScreenShare`, render a `<video>` element filling a larger tile area (2x size in grid). No avatar — just the screen content.
