**startScreenShare() implementation:**

1. Call `navigator.mediaDevices.getDisplayMedia({ video: true, audio: false })`
   - This shows the browser's screen picker (user selects window/screen)
   - Returns a MediaStream with the selected content

2. For each track in the screen stream:
   - For each peer connection: `pc.addTrack(track, screenStream)`
     - This triggers `negotiationneeded` → perfect negotiation sends new offer
   - Store senders for cleanup

3. Apply bandwidth constraint for 4+ participants (same as camera: 200kbps limit)

4. Listen for `track.onended` (fires when user clicks browser's "Stop sharing" button — Pitfall 4):
   ```typescript
   track.onended = () => {
     // Remove track from all peer connections
     for (const pc of peersRef.current.values()) {
       const sender = pc.getSenders().find(s => s.track === track);
       if (sender) {
         pc.removeTrack(sender); // triggers renegotiation
       }
     }
     // Clean up local state
     localScreenSharesRef.current.delete(streamId);
     setScreenShares(prev => {
       const next = new Map(prev);
       next.delete(streamId);
       return next;
     });
     // Broadcast
     socket.emit("voice:screen_share", {
       channelId,
       screenShareCount: localScreenSharesRef.current.size,
       streamId,
       action: "stopped"
     });
   };
   ```

5. Update state and broadcast:
   ```typescript
   localScreenSharesRef.current.set(screenStream.id, screenStream);
   setScreenShares(new Map(localScreenSharesRef.current));
   socket.emit("voice:screen_share", {
     channelId,
     screenShareCount: localScreenSharesRef.current.size,
     streamId: screenStream.id,
     action: "started"
   });
   ```

**stopScreenShare(streamId):**
- Stop all tracks in the stream
- Remove senders from all PCs (triggers renegotiation)
- Clean up maps and state
- Broadcast `voice:screen_share` with action "stopped"

**Cleanup on leave:** Stop all screen share tracks, remove all senders, clear maps.
