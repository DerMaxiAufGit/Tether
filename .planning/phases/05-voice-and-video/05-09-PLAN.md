The camera toggle works for remote peers (track sent via RTCPeerConnection) but the self-view tile never shows video because:
1. `toggleCamera()` stores the camera track in `cameraTrackRef` but never adds it to `localStream` (which is audio-only)
2. `ParticipantGrid` passes `localStream` to the self-tile
3. `ParticipantTile.hasVideo` checks `stream.getVideoTracks()` which is empty on the audio-only stream

Fix: expose a separate `localCameraStream` in the hook state. When camera turns on, create `new MediaStream([videoTrack])` and store it in state. When camera turns off, set it to null. The new stream reference triggers a React re-render in ParticipantTile so `hasVideo` recomputes.

**Step 1 — useVoiceChannel.ts:**

Add `localCameraStream: MediaStream | null` to VoiceState interface (default: null).

In `toggleCamera()`, when camera ON:
```typescript
// After setting cameraTrackRef.current = videoTrack:
const cameraStream = new MediaStream([videoTrack]);
setState((prev) => ({ ...prev, cameraOn: true, localCameraStream: cameraStream }));
```

When camera OFF:
```typescript
setState((prev) => ({ ...prev, cameraOn: false, localCameraStream: null }));
```

In `cleanupAll()`, clear localCameraStream:
```typescript
// Already stops cameraTrackRef — also clear state
```
Note: cleanupAll already sets all state to defaults via leave() which resets the full state object. Just ensure `localCameraStream: null` is in the reset state object in `leave()`.

In the return object, add `localCameraStream: state.localCameraStream`.

**Step 2 — VoiceContext.tsx:**
No changes needed — the context type is `ReturnType<typeof useVoiceChannel>` which auto-includes new fields.

**Step 3 — ParticipantGrid.tsx:**

Add `localCameraStream: MediaStream | null` to ParticipantGridProps.

For the self-tile, pass `localCameraStream ?? localStream` as the `stream` prop instead of just `localStream`. This way the self-tile receives the camera stream when camera is on, and the audio-only stream otherwise.

```tsx
{/* Self participant tile */}
{(() => {
  const self = participants.find((p) => p.userId === selfUserId);
  if (!self) return null;
  return (
    <ParticipantTile
      key={`self-${selfUserId}`}
      participant={self}
      stream={localCameraStream ?? localStream}
      isSelf
    />
  );
})()}
```

**Step 4 — VoiceChannelView.tsx:**

Pass `voice.localCameraStream` to ParticipantGrid:

No changes needed to ParticipantTile — the new stream reference is sufficient to trigger re-render and recompute hasVideo.
